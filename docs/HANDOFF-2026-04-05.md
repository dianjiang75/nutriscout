## GUARDRAILS ADDED (to prevent recurrence)

### 1. Backup System
```bash
# MUST run before ANY destructive operation
./scripts/backup-db.sh [label]
# Restore: pg_restore -d "$DATABASE_URL" --clean backups/FILENAME.dump
```

### 2. Memory Rules (saved in memory files)
- **NEVER run `seed-manhattan.ts`** on a DB with crawled data — it deletes everything
- **NEVER use `deleteMany()` without a WHERE clause** targeting specific items
- **ALWAYS backup before delete** — no exceptions
- **Test cleanup scripts with dry-run first**
- File: `.claude/projects/.../memory/feedback_never_delete_data.md`

### 3. Seed Script Warning
`scripts/seed-manhattan.ts` runs `deleteMany()` on ALL tables. It should NEVER be used as a "restore" mechanism. For future sessions: if dishes get incorrectly deleted, restore from backup — don't re-seed.
