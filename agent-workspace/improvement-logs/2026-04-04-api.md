# API Agent Report — 2026-04-04

## Changes Made

### 1. Rate Limiting — 4 Routes Protected
Added `withRateLimit("read", ...)` wrapper to previously unprotected routes:

- **`/api/restaurants`** (GET) — was wide open, now rate-limited to 100 req/min
- **`/api/restaurants/[id]`** (GET) — was wide open, now rate-limited
- **`/api/restaurants/[id]/menu`** (GET) — was wide open, now rate-limited
- **`/api/dishes/[id]`** (GET) — was wide open, now rate-limited

### 2. Consistent Error Responses — `/api/restaurants`
- **Fix**: Replaced `Response.json({ error: ... })` with `apiBadRequest()` and `apiError()`
- **Fix**: Added coordinate range validation (lat: -90 to 90, lng: -180 to 180)
- **Impact**: All error responses now use consistent `{ success: false, error: ... }` envelope

### 3. Input Validation — `/api/restaurants`
- **Fix**: Added bounds checking for lat/lng coordinates
- **Why**: Nonsensical coordinates (e.g., lat=999) were accepted and would cause earthdistance to return garbage
