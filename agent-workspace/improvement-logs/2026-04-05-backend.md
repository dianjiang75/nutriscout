## Changes Made

### 1. Tag-Based Cache Invalidation (src/lib/cache/index.ts)

**Before:** `invalidateRestaurant()` used Redis SCAN to find keys matching `restaurant:{id}:*`. SCAN is O(N) on the entire keyspace — fine at 100 keys, problematic at 100K+.

**After:** Tag-based invalidation using Redis SETs:
- `cacheTagAdd(tag, key)` — registers a cache key under a tag SET (`cache-tag:{tag}`)
- `cacheSetTagged(key, value, domain, tags)` — convenience wrapper that sets + tags in one call
- `invalidateRestaurant(id)` — now uses `SMEMBERS` + pipeline `DEL` instead of SCAN. O(M) where M = keys per restaurant (typically < 20)
- `invalidateTag(tag)` — generic tag invalidation for any use case
- `invalidateAllQueries()` — tries tag-based first (`queries` tag), falls back to SCAN for legacy untagged keys
- `setCachedQuery()` now registers each query key under the `queries` tag

**Complexity:** O(M) per invalidation vs O(N) keyspace scan. Tags auto-expire at 31 days.
