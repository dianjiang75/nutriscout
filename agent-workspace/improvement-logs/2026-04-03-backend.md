## Changes Applied

### 1. Post-migrate.sql — 3 new indexes (MEDIUM)
**File**: `scripts/post-migrate.sql`
**Added**:
- `idx_dishes_macro_source` — composite on (macro_source, created_at DESC) for sorting dishes by data quality
- `idx_delivery_last_checked` — on restaurant_deliveries.last_checked for stale delivery data detection
- `idx_dishes_low_confidence` — partial index on macro_confidence < 0.7 for finding dishes needing re-analysis

### 2. Redis eager connect (MEDIUM)
**File**: `src/lib/cache/redis.ts`
**Problem**: `lazyConnect: true` defers TCP handshake until first Redis command, so misconfigurations (wrong URL, auth failure) don't surface until a user hits an endpoint.
**Fix**: Call `redis.connect()` eagerly after creation. The promise is fire-and-forget (errors handled by the existing "error" listener), but it initiates the connection immediately.

### 3. Rate limiter member uniqueness (LOW)
**File**: `src/lib/middleware/rate-limiter.ts`
**Problem**: Sorted set member uses `${now}:${Math.random()}` — `Math.random()` can theoretically collide under high concurrency (same timestamp + same random value = ZADD overwrites instead of adding).
**Fix**: Replaced with monotonic counter `${now}:${++memberCounter}` which is guaranteed unique within a process.

### 4. Logger production level (LOW)
**File**: `src/lib/utils/logger.ts`
**Problem**: Production min level was `warn`, making normal API request logs invisible in monitoring.
**Fix**: Changed default to `info` for production, configurable via `LOG_LEVEL` env var.
