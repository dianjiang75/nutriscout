# Backend Agent Report — 2026-04-04

## Changes Made

### 1. Post-Migrate SQL Indexes (`prisma/post-migrate.sql`) — NEW FILE
Critical indexes Prisma can't express natively:
- **GIN on dietaryFlags** (jsonb_path_ops): 36x smaller, fast `@>` containment queries for dietary filtering
- **GIN on search_vector** (english): Full-text search on dish name + description
- **GIN on search_vector_simple**: Non-English dish names (ramen, pho, bibimbap)
- **GIN trgm on name**: Fuzzy name matching via `similarity()` threshold
- **GiST on restaurant geo**: earthdistance `ll_to_earth()` for geo pre-filter index scan
- **Partial index on low confidence**: Target dishes for re-analysis
- **Partial index on delivery staleness**: Find stale delivery data
- **HNSW on embedding** (if exists): ef_construction=128 for better recall

### 2. Cache Error Handling (`src/lib/cache/index.ts`)
- **Fix**: Wrapped `cacheGet`, `cacheSet`, `cacheDel` in try/catch
- **Why**: Redis failures were uncaught — propagated to API responses as 500 errors
- **Impact**: Cache failures now fall through gracefully (cache miss → DB query)

### 3. Logger Correlation IDs (`src/lib/utils/logger.ts`)
- **Fix**: Added `requestId` field, generated from `x-request-id` header or `crypto.randomUUID()`
- **Fix**: Added slow request detection (> 2s) in production
- **Impact**: Can trace requests across logs; slow queries visible in prod monitoring

### 4. Slow Query Logging in Production (`src/lib/db/client.ts`)
- **Fix**: Enabled query event logging and slow query detection in all environments (was dev-only)
- **Fix**: Threshold configurable via `SLOW_QUERY_MS` env var (default 500ms)
- **Impact**: Slow queries no longer invisible in production
