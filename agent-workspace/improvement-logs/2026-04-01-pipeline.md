## Changes Made

### 1. Price parsing robustness (`menu-crawler/index.ts`)
- **Before**: `parseFloat(raw.price.replace(/[^0-9.]/g, ""))` ‚Äî "$10 - $15" became 10.15, "Market Price" became NaN
- **After**: New `parsePriceString()` handles ranges (takes lower bound), "Market Price", "$$$" indicators
- **Impact**: Prevents corrupted price data in DB

### 2. Review summarization JSON parsing (`review-aggregator/index.ts`)
- **Before**: `JSON.parse(textBlock.text)` ‚Äî fails when LLM returns markdown-fenced JSON
- **After**: Uses `extractJson<DishReviewSummary>()` which handles fences, leading/trailing text
- **Impact**: Prevents silent review summarization failures

### 3. Dish deduplication on re-crawl (`menu-crawler/index.ts`)
- **Before**: `prisma.dish.create()` ‚Äî re-crawling creates duplicate dishes
- **After**: `findFirst` + `update` / `create` pattern prevents duplicates
- **Impact**: Clean data on repeated crawls

### 4. Failed batch ingredient analysis recovery (`menu-crawler/index.ts`)
- **Before**: Failed JSON parse silently drops entire batch
- **After**: Returns placeholder entries with `dietary_confidence: 0` and warning message
- **Impact**: Callers know which items need manual review; no silent data loss

### 5. Vision batch analysis concurrency (`vision-analyzer/index.ts`)
- **Before**: Sequential `for` loop processing photos one at a time
- **After**: Processes 3 photos concurrently with `Promise.allSettled`
- **Impact**: ~3x faster batch photo analysis

### 6. USDA synonym mapping (`usda/client.ts`)
- **Before**: Raw ingredient names like "shrimp" searched as-is (poor USDA matches)
- **After**: 30+ common ingredient synonyms map to USDA-friendly terms (e.g. "shrimp" ‚Üí "shrimp, cooked")
- **Impact**: Better USDA match quality for common ingredients
