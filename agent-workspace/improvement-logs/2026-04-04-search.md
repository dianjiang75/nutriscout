# Search Agent Report — 2026-04-04

## Changes Made

### 1. FTS Simple Dictionary — Use Indexed Column (`src/lib/db/geo.ts`)
- **Fix**: Changed simple dictionary fallback to use `search_vector_simple` generated column + GIN index instead of on-the-fly `to_tsvector('simple', name)`
- **Why**: On-the-fly tsvector computation caused sequential scans on every fallback query. The `search_vector_simple` column is now defined in post-migrate.sql with a GIN index.
- **Impact**: Foreign dish name searches (ramen, pho, bibimbap) now use index scan instead of seq scan

### 2. Diversity Cap Re-Sort (`src/lib/orchestrator/index.ts`)
- **Fix**: Added re-sort for `macro_match` and default relevance after diversity cap
- **Why**: The cap removes items from the sorted list, which can break ordering. Rating/distance/wait_time were re-sorted, but macro_match and default relevance were not — causing Best Match results to lose their relevance ordering after cap.
- **Impact**: Search results maintain correct ordering in all sort modes

### 3. DB-Calculated Distances (`src/lib/orchestrator/index.ts`)
- **Fix**: Use distances from `getRestaurantIdsWithinRadius()` (earthdistance) instead of recalculating haversine
- **Why**: earthdistance is more accurate for short distances; eliminates redundant calculation; distances already available from geo pre-filter
- **Impact**: Slight accuracy improvement + eliminates redundant computation
