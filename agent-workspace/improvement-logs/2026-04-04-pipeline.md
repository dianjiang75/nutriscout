## Changes Made

### 1. USDA Portion Validation (`src/lib/usda/client.ts`)
- **Fix**: Added portion clamping (1g–5000g) in `estimateMacros()`
- **Why**: Vision analyzer can return 0g or extremely large portions on errors, causing nonsensical macro values (0 cal or 500,000 cal)
- **Impact**: Prevents garbage data propagation to search results

### 2. Crawl Worker Photo Cap (`workers/crawl-worker.ts`)
- **Fix**: Added `MAX_PHOTOS_PER_CRAWL = 20` limit
- **Why**: Large restaurants with 50+ dishes could flood the photo queue, creating backlog
- **Impact**: Prevents queue starvation; prioritized dishes still analyzed first

### 3. Photo Worker Input Validation (`workers/photo-worker.ts`)
- **Fix**: Early return if `dishId` or `photoUrl` is missing
- **Why**: Invalid job data wastes API calls and generates confusing errors
- **Impact**: Cleaner error handling, faster job processing

### 4. Menu Crawler Logging (`src/lib/agents/menu-crawler/index.ts`)
- **Fix**: Added contextual logging for photo job queueing (restaurant name, count)
- **Why**: When photo analysis fails, need to know which restaurant was affected
- **Impact**: Better debugging for nightly pipeline runs
