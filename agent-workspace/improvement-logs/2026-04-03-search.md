## Changes Applied

### 1. Evaluator — Eggs mapped to dairy_free flag (CRITICAL SAFETY)
**File**: `src/lib/evaluator/index.ts:42`
**Problem**: `eggs: "dairy_free"` in ALLERGEN_TO_FLAG. Eggs are NOT dairy products. A user excluding eggs would have their exclusion checked against the `dairy_free` flag, which is incorrect. An egg-containing dish that is dairy-free would pass the check when it shouldn't.
**Fix**: Removed the `eggs: "dairy_free"` mapping. Egg allergen now uses keyword-only matching (the `ALLERGEN_KEYWORDS.eggs` list), which is correct since there's no `egg_free` dietary flag.

### 2. Wait Time Filter — Strict < vs Inclusive <=  (MEDIUM)
**File**: `src/lib/orchestrator/index.ts:283`
**Problem**: Used `wait < query.max_wait_minutes` (strict less-than). User expectation: "max 15 min" means 15 is OK but 16 is not.
**Fix**: Changed to `wait <= query.max_wait_minutes`.

### 3. Evaluator — Redundant Allergen Keyword Checks (LOW)
**File**: `src/lib/evaluator/index.ts:120-132`
**Problem**: Three sequential checks all testing `keywords.some(kw => desc.includes(kw))`:
  - Line 122: if critical + keywords found → return false
  - Line 126: if flag !== true + keywords found → return false
  - Line 130: if flag === true + keywords found → return false
  All three returned false on keyword match, making the latter two unreachable after the first.
**Fix**: Consolidated into a single keyword check that excludes on any match, plus a clean non-critical flag fallthrough.
